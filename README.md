# Pattaya Kids Guide

*A calm, verified live guide for Russian-speaking families in Pattaya, Thailand.*

Спокойный живой гид для родителей в Паттайе: **места** (куда пойти с ребёнком сейчас), **события** (что происходит) и **занятия** (куда записать — студии, развивашки, лагеря). Не каталог-свалка, а проверенные руками данные с честным «уточняется» там, где факт ещё не подтверждён.

Интерфейс — на русском (аудитория — русскоязычные родители-резиденты). Код, схема и slug'и — на английском.

---

## Что уже умеет

**Сценарные фильтры** — кнопки от жизненных ситуаций родителя, а не от типов заведений:

- **«Пойти сейчас»** — открыто в эту минуту или откроется в ближайшие 30 минут;
- **«Открыто с утра»** — сегодня открывается к 9:00;
- **«Можно поработать»** — Wi-Fi + кондиционер + где посидеть с ноутбуком;
- **«Спрятаться от жары/дождя»** — в помещении с кондиционером *или* под навесом с вентиляторами.

**Живой статус** — «Открыто ещё ~5 часов» / «Скоро закрытие» / «Откроется в 11:00» считается на каждый запрос по расписанию места и часовому поясу города; открытые места поднимаются выше закрытых. Тот же язык статусов у событий («Сейчас идёт» / «Уже прошло»).

**Честность как принцип.** Факты о месте — трёхзначные: `true` (есть) / `false` (точно нет) / `null` («уточняется» на сайте). Пробел в данных никогда не выдаётся за уверенное «нет». Советы «Полезно знать» несут дату проверки («проверено: июль 2026»). Чек-лист несобранных данных: `npm run gaps`.

**Страница места** — порядок «как родитель думает»: сводка-чипы в шапке (возраст · вход от · можно оставить · сегодня до) → фото → часы → цены (включая табличку «1/3/5 часов × ребёнок/взрослый» — как прайс на стойке) → советы → занятия/абонементы → день рождения → факты → адрес и контакты.

**Двухосевой продукт:** «Места» (спонтанно: куда пойти) и «Занятия» (планово: куда записать) — два входа в одни данные. У занятия своя страница с расписанием классов («с родителем / без родителя» по возрастам), фильтры по возрасту ребёнка и типу занятий. Занятие может существовать без каталожного места (развивашка на базе сада).

**Ещё:** пакеты дней рождения с реальными ценами, контакты-пилюли (LINE/Instagram/телефон), кнопка «Поделиться», мини-галереи, своя 404, sitemap с SEO-гейтом (город не индексируется, пока не наполнен реальными местами).

## Статус проекта

Пре-MVP в активной разработке. Модель данных заложена «широко» (мультигород, UGC, модерация, монетизация), наполнение — «узко»: Паттайя, реальные места заносятся вручную по личным визитам. Публичного деплоя пока нет. Актуальные планы — в [docs/](docs/): [ROADMAP.md](docs/ROADMAP.md), [MULTIREGION_PLAN.md](docs/MULTIREGION_PLAN.md), [SMART_FILTERS_PLAN.md](docs/SMART_FILTERS_PLAN.md).

Файлы `AI_CONTEXT.md`, `PROJECT_MAP.md`, `NEXT_STEPS.md` в корне — исторические, коду не соответствуют; источник истины — код, `git log` и `docs/`.

## Стек

- **Next.js 16** (App Router, SSR) + **React 19** + **TypeScript** (strict)
- **Prisma 7** + `@prisma/adapter-pg` + **PostgreSQL** (пул соединений — обязателен adapter-клиент, см. `src/db/prisma.ts`)
- Схема применяется через `prisma db push` — **миграций в проекте нет**
- Стили — обычный CSS (`src/app/globals.css`), calm-first палитра: тёплый беж, белые карточки, приглушённые sage/янтарь/терракота; никакого чистого чёрного и кислотных цветов

## Архитектура

```
route (страница/`app/api`) → service → mapper → DTO
```

- Страницы (`src/app/[lang]/[city]/…`) вызывают сервисы **напрямую, in-process** — REST-роуты в `src/app/api/` фронтендом не используются (задел под будущего Telegram-бота).
- Роутинг мультирегиональный: `/{lang}/{city}/…` (сейчас `/ru/pattaya`, `/` делает redirect). Слаги мест/событий уникальны в пределах города.
- Все видимые тексты — в одном словаре `src/content/ru.ts`.
- Вычислимое (статус «открыто сейчас», жизненный цикл событий, возрастные корзины) — чистые функции в `src/lib/…` с порогами-константами.

```
src/
  app/[lang]/[city]/   страницы: места (корень), events, activities + детали
  app/api/             REST (не используется фронтом)
  components/          карточки, фильтры, сценарии, статусы
  services/            запросы к БД (Prisma)
  mappers/ + dto/      Prisma-модель → публичная форма
  lib/                 schedule/, events/, activities/, geo/, demo/ …
  content/ru.ts        все тексты интерфейса
prisma/
  schema.prisma        модель данных (широкая: гео, UGC, модерация)
  seed.ts              идемпотентный сид (upsert), реальные места + [Демо]
  gaps.ts              npm run gaps — чек-лист несобранных данных
```

## Запуск

```bash
npm install

# .env (не коммитится):
#   DATABASE_URL=postgresql://…   (без кавычек)
#   SHOW_DEMO="true"              (локально: показывать [Демо]-контент; на деплое не задавать)

npx prisma db push        # применить схему (миграций нет)
npx prisma generate       # пересобрать клиент (db push его НЕ пересобирает)
npx tsx prisma/seed.ts    # идемпотентный сид

npm run dev               # http://localhost:3000 → /ru/pattaya
```

Полезные скрипты: `npm run check` (lint + typecheck), `npm run gaps` (что осталось собрать по местам), `npm run fix` (prettier + eslint --fix).

⚠️ После правки `schema.prisma` всегда: `db push` → `generate` → перезапуск dev-сервера (запущенный сервер держит старый клиент).

## Принципы

- **Беречь видение**: спокойный тон, «UI тише контента», нейтральность к обоим родителям. Если продукт превращается в generic-каталог — мы свернули не туда.
- **Не выдумывать данные**: неизвестное — это `null` и «уточняется», а не правдоподобное значение.
- **Тонкие обработчики**, бизнес-логика в сервисах; Prisma-модели наружу не течут — только DTO.
- **Малые коммиты** (Conventional Commits), пуш после каждого логического куска.
- Прошедшие события **не удаляются** (архив, SEO, доверие).
- Читаемость важнее «умного» кода; без `any`, без over-engineering.

Этот проект — больше, чем код:

- шаг к финансовой самостоятельности.

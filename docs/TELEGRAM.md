# Telegram-интеграция

Сайт связан с Telegram двумя способами:

1. **Автопостинг в канал** — новые одобренные события и места публикуются постами в канал (`TELEGRAM_CHANNEL_ID`). Запускается кроном Vercel раз в день в 10:07 по Паттайе (см. `vercel.json`) или вручную: `npm run telegram:autopost`.
2. **Бот-помощник** — отвечает людям в личке подборками: `/today`, `/weekend`, `/soon`, `/places` (плюс кнопки). Сообщения приходят на вебхук `/api/telegram/webhook`.

Данные оба берут из той же базы (Neon), что и сайт. Ссылки в постах ведут на страницы сайта.

## Карта файлов

| Файл                                           | Что делает                                        |
| ---------------------------------------------- | ------------------------------------------------- |
| `src/lib/telegram/client.ts`                   | Тонкий клиент Bot API (fetch, без зависимостей)   |
| `src/lib/telegram/format.ts`                   | Тексты постов и подборок, даты по Паттайе, ссылки |
| `src/lib/telegram/types.ts`                    | Типы Telegram-апдейтов и клавиатур                |
| `src/services/telegram-autopost.service.ts`    | Логика автопостинга + журнал дублей               |
| `src/services/telegram-bot.service.ts`         | Логика ответов бота                               |
| `src/app/api/telegram/webhook/route.ts`        | Вебхук для сообщений боту                         |
| `src/app/api/telegram/autopost/route.ts`       | Endpoint для крона Vercel                         |
| `scripts/telegram/*.ts`                        | Ручные команды (см. ниже)                         |
| Модель `TelegramPost` в `prisma/schema.prisma` | Журнал «что уже опубликовано»                     |

Архитектура сайта соблюдена: route-обработчики тонкие, бизнес-логика в сервисах, наружу отдаются DTO (переиспользуется `mapEventListItemToDto`).

## Переменные окружения

Локально — в `.env` (без кавычек!), на Vercel — Settings → Environment Variables.

| Переменная                | Что это                                               | Где взять                                                   |
| ------------------------- | ----------------------------------------------------- | ----------------------------------------------------------- |
| `TELEGRAM_BOT_TOKEN`      | Ключ от бота. Секрет!                                 | BotFather → `/mybots` → бот → API Token                     |
| `TELEGRAM_CHANNEL_ID`     | Куда постить                                          | `@pattayakidsguide`                                         |
| `TELEGRAM_WEBHOOK_SECRET` | Подпись, по которой вебхук отличает Telegram от чужих | генерирует установщик                                       |
| `CRON_SECRET`             | Пароль крона Vercel к autopost-endpoint               | генерирует установщик; Vercel сам шлёт его в запросах крона |
| `NEXT_PUBLIC_SITE_URL`    | Адрес сайта для ссылок в постах                       | напр. `https://<проект>.vercel.app`, без `/` в конце        |

## Команды

```bash
npm run telegram:test-post      # тестовый пост в канал (проверка токена и прав)
npm run telegram:baseline       # ОДИН РАЗ при подключении: пометить весь текущий
                                #   каталог как «уже опубликовано», без отправки
npm run telegram:autopost:dry   # показать, что ушло бы в канал (без отправки)
npm run telegram:autopost       # опубликовать новое прямо сейчас
npm run telegram:webhook        # привязать вебхук бота к сайту (после деплоя)
```

## Первый запуск

1. `node install.mjs` из архива уже разложил файлы, добавил модель в схему, скрипты в package.json и заготовки в `.env`.
2. Вставь токен бота в `.env` → `TELEGRAM_BOT_TOKEN=...` (и проверь `NEXT_PUBLIC_SITE_URL`).
3. `npx prisma db push` — создаст таблицу журнала в базе.
4. `npm run telegram:test-post` — в канале появится тестовый пост.
5. По желанию: `npm run telegram:baseline` — помечает ВЕСЬ текущий каталог как «уже опубликовано», и канал начнёт с чистого листа. Если каталог пока маленький и хочется стартовых постов — пропусти этот шаг: автопостинг сам опубликует текущие места (не больше 5 за прогон).
6. Закоммить и запушить изменения — Vercel задеплоит.
7. На Vercel добавь все 5 переменных (значения из `.env`) и сделай Redeploy.
8. `npm run telegram:webhook` — бот оживёт: напиши ему `/start`.

## Ежедневная жизнь

- Добавила события/места в базу → `npm run telegram:autopost` (или просто жди: крон опубликует утром в 10:07).
- За один прогон уходит максимум 5 постов — очередь дойдёт за следующие дни, канал не заспамится.
- Постятся только: события со статусом `APPROVED` и датой в будущем, места со статусом `APPROVED`.

## Если что-то не работает

- **401 Unauthorized** — неверный токен: проверь `TELEGRAM_BOT_TOKEN`.
- **403 / not enough rights** — бот не админ канала или без права «Публикация сообщений».
- **Бот молчит в личке** — проверь вебхук: `npm run telegram:webhook` покажет `last_error_message`; переменные на Vercel добавлены? был Redeploy?
- **Пост без картинки** — `imageUrl` битый или файла нет в `public/`; пост уходит текстом (это осознанный fallback).
- **Дубль поста** — невозможен по журналу; если пост удалили из канала руками, повторно он не выйдет (запись в `TelegramPost` осталась).

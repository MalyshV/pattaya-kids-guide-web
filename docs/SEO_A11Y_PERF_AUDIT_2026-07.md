## TL;DR — пять главных выводов

1. **«Долго открывается» — это холодные старты, а не тяжёлый фронт.** Прогретый
   сервер отвечает за ~170 мс. Но каждый запрос рендерится заново (`no-store`,
   кэша нет нигде), поэтому «первый» визит платит полную цепочку: пробуждение
   serverless-функции + пробуждение уснувшей Neon-БД + полный SSR. Лечится
   ISR + кэшем данных + `loading.tsx`.
2. **Сайт закрыт от индексации собственным SEO-гейтом** — город в прод-базе
   `isPublished: false`, отсюда noindex на всех страницах и пустой sitemap.xml.
   Это правильный механизм, но включать индексацию можно только после починки
   пункта 3.
3. **hreflang на детальных страницах сломан**: карточки мест/событий/занятий
   наследуют языковые ссылки на корень каталога и не имеют canonical. Для
   мультиязычного сайта это самый серьёзный SEO-дефект.
4. **`<html lang="ru">` на тайских и английских страницах** (правится только
   клиентским JS после гидрации). Скринридер читает тайский текст русским
   синтезатором — для тайской аудитории сайт фактически недоступен. Главный
   a11y-приоритет.
5. **Терракотовый акцент `#c96f4a` не дотягивает по контрасту** (3.0–3.6:1 при
   норме 4.5), а прозрачность на карточках закрытых мест роняет весь их текст
   до 3.4:1 — именно вечером, когда родители планируют завтра. Лечится вторым
   токеном той же терракоты потемнее — палитра и атмосфера не меняются.

## Почему сайт долго открывается (разбор)

Замеры прода (регион function и БД — Сингапур, до Паттайи близко):

| Что | Значение |
| --- | --- |
| TTFB прогретой страницы | 0.17–0.24 с (норм) |
| Кэш Vercel | `no-store`, `x-vercel-cache: MISS` на каждом запросе |
| LCP (Lighthouse, эмуляция слабого телефона) | 3.0 с |
| Speed Index (там же) | 5.1 с |

Цепочка медленного «первого» визита:

1. **Ни одна публичная страница не кэшируется.** Во всём `src/app` нет
   `generateStaticParams`, `revalidate` или `unstable_cache`. Списки динамичны
   из-за `searchParams` (`src/app/[lang]/[city]/page.tsx:72`), детальные —
   просто без `generateStaticParams`. Каждый визит = запуск функции + 2–5
   запросов к БД до первого байта.
2. **Neon автосуспендит БД после простоя** (бесплатный план), пробуждение —
   0.5–3 с. При малом трафике почти каждый заход Вероники будит и функцию,
   и базу — обе задержки складываются в TTFB.
3. **Нет ни одного `loading.tsx`** — пока сервер ждёт БД, пользователь смотрит
   на белый экран/старую страницу. Layout города сам ходит в БД до рендера
   (`src/app/[lang]/[city]/layout.tsx:60`), так что не стримится даже шапка.

Что **не** виновато (проверено): middleware нет; tesseract.js — только в
админке и лениво; leaflet — динамическим импортом; картинки идут через
`next/image` с корректным `sizes` (`src/components/places/place-image.tsx:30`);
N+1-запросов нет; Prisma — корректный синглтон с pooled-endpoint; регион
функции и БД совпадают.

## SEO

### Критично (чинить ДО включения индексации)

- **Детальные страницы: нет canonical, hreflang указывает на чужую страницу.**
  `generateMetadata` карточек не задаёт `alternates`, и они наследуют
  `alternates.languages` из city-layout — т.е. hreflang каждой карточки ведёт
  на `/ru/pattaya`, а не на переводы самой карточки (подтверждено на проде).
  Фикс: в `generateMetadata` детальных добавить `alternates` по образцу
  `listPageAlternates` (`src/lib/seo/meta.ts`), с `subPath = "/places/${slug}"`.
  Файлы: `places/[slug]/page.tsx:33-67`, `events/[slug]/page.tsx:24-57`,
  `activities/[slug]/page.tsx:23-61`.
- **Sitemap строится только для `ru` и без занятий.** `src/app/sitemap.ts:30`
  использует `DEFAULT_LANG`; `activities/[slug]` не выбираются вовсе. Фикс:
  генерировать записи для всех `SUPPORTED_LANGS` (в идеале — с
  `alternates.languages` в записях), добавить занятия.
- **`<html lang>` серверно всегда `ru`** — общий пункт с доступностью, см. ниже.
- **Нет `x-default` в hreflang** (`[lang]/[city]/layout.tsx:36-40`,
  `seo/meta.ts:25-27`). Добавить `"x-default"` → дефолтная локаль.

Контекст: текущий noindex и пустой sitemap — работа SEO-гейта
(`[lang]/[city]/layout.tsx:42` + `sitemap.ts:14`): город в прод-базе не
опубликован. Выключатель на месте; последовательность запуска — сначала
четыре пункта выше, потом `isPublished: true`.

### Важно

- **JSON-LD отсутствует полностью.** Для гида это большой недобор — данные уже
  структурированы в БД. Порядок внедрения: `Event` (даты, место → право на
  event rich results), `Place`/`LocalBusiness` (адрес, geo,
  openingHoursSpecification), `BreadcrumbList`, `WebSite + SearchAction`.
- **Нет дефолтного og:image** — превью ссылки без картинки у списков и карточек
  без фото (комментарий-признание в `layout.tsx:37-46`). Главный канал роста —
  «поделиться в чат», так что это прямая потеря. Добавить
  `src/app/opengraph-image.png` или генерацию через `ImageResponse`.
- **Прод на `*.vercel.app`.** Купить кастомный домен до контентного роста,
  иначе позже — 301-миграция всего индекса. Vercel сам повесит редирект.
- **`getSiteUrl()` молча падает на localhost**, если `NEXT_PUBLIC_SITE_URL`
  потеряется (`src/lib/geo/city.ts:19-21`; в `.env.example` переменная
  закомментирована). Фикс: fallback на `VERCEL_PROJECT_PRODUCTION_URL`.

### Мелочи

- Title каталога — «бренд-первый», без запроса («детские места», «куда пойти
  с ребёнком») — у birthdays сделано правильно, распространить паттерн.
- `title.template` в root layout вместо ручного `— ${dict.brand}` на страницах.
- `lastModified: new Date()` у разделов в sitemap — заменить на реальные даты
  или убрать.
- Редирект `/` — 307; когда дефолт станет финальным, сделать 308.
- Canonical пагинации указывает на стр. 1 — приемлемый компромисс, знать.
- `pickLocalized` может отдавать русский текст под en/th hreflang — следить,
  чтобы ключевые страницы имели реальные переводы (для th уже сделано).

### Что уже хорошо (SEO)

robots.ts + sitemap.ts подключены; metadataBase задан; SEO-гейт городов —
отличная идея на вырост; self-canonical на списках гасит дубли фильтров
(`?openNow=`, `?age=`, `?view=map`, `?page=` — проверено на проде);
generateMetadata на каждом маршруте, description режется по слову до 160;
OG на карточках с фото; noindex только там, где надо (админка, «Избранное»);
локаль в пути (клоакинга нет); честные 404 (слаг/город/язык — проверено,
отдаёт 404-статус); один h1 на страницу; карточки — настоящие ссылки;
демо-контент отсечён из выдачи и sitemap.

## Доступность (WCAG 2.2 AA)

### Критично

- **`<html lang="ru">` на en/th-страницах** (`src/app/layout.tsx:60`; клиентский
  `HtmlLangSync` правит после гидрации). Скринридер читает тайский/английский
  русским голосом; без JS — навсегда (WCAG 3.1.1). Для тайской аудитории —
  то же, что отсутствие сайта. Фикс: рендерить `<html>` в layout, знающем
  `params.lang` (корневой layout внутрь сегмента `[lang]`; `/admin` — в route
  group со своим layout). `HtmlLangSync` оставить как страховку клиентских
  переходов. Это одна правка, закрывающая и SEO-пункт.

### Важно

- **Контраст терракоты `#c96f4a`** (норма 4.5:1 для текста; значения
  подтверждены Lighthouse на проде):
  - белый текст на терракотовой заливке = **3.59** — `.primary-button`,
    `.filter-toggle-active`, `.filter-chip-active`, `.empty-state-cta`,
    `.saved-empty-cta`, `.site-nav-saved-count`, `.admin-button`;
  - терракотовый текст на светлом = **3.52/3.24** — `.place-card-cta`
    («Подробнее»), `.back-link`, `.map-link`, `.eyebrow`, `.program-type`,
    ссылки в `hero-venue` и попапе карты.
  - Фикс в духе палитры: второй токен **`--accent-text: #a65136`** (та же
    тёплая терракота глубже: 5.37:1 на карточке, 4.93:1 на беже) для текста;
    заливку кнопок затемнить до `#a9563a` (5.15:1 с белым). Декоративная
    терракота (точки, рамки, ♥) остаётся как есть — для графики 3:1 соблюдено.
- **Прозрачность закрытых карточек убивает контраст.** `opacity: 0.8`
  (`globals.css:1530-1537`) превращает `--muted` в фактический `#867e74` —
  адреса и чипы падают до **3.4:1**. Подтверждено Lighthouse: вечером, когда
  большинство мест закрыто, почти все карточки каталога проваливают контраст —
  ровно в час, когда родитель планирует завтрашний день. Фикс: вместо opacity
  задавать явные приглушённые цвета (не светлее `#6b6358`); то же для
  `.tip-verified` (`opacity: 0.75`).
- **«Закрыто» в часах работы** — muted + `opacity: 0.65` = **2.78:1**
  (`globals.css:2411-2413`). Убрать opacity — чистый `--muted` даёт 5.8:1.
- **Результаты фильтрации обновляются молча** для скринридеров: счётчик
  «Найдено N мест» (`page.tsx:307`) без `role="status"` (WCAG 4.1.3). Одна
  строка — и незрячий пользователь слышит, что фильтр сработал. Точечные
  статусы уже есть (`places-results.tsx:321-331`) — распространить.
- **Нет skip-link** — 8+ ссылок шапки на каждой странице до контента
  (WCAG 2.4.1; подтверждено табом по проду). Визуально скрытая ссылка
  `href="#main"`, появляющаяся по фокусу.
- **Кнопки «×» в Избранном неразличимы**: одинаковый aria-label «Убрать» у
  всех (`saved-list.tsx:139`). Фикс: `${dict.memory.remove}: ${item.name}` —
  по образцу уже сделанного в `memory-buttons.tsx:44`.
- **Триггер меню языка**: видимый текст «RU» не входит в accessible name
  (`site-header.tsx`, кнопка `.lang-menu-trigger`; поймано Lighthouse
  label-content-name-mismatch). Включить видимый текст в aria-label
  («Язык: RU») — иначе голосовое управление командой «нажми RU» не работает.

### Мелочи

- «Скоро закрытие» `#97612f/#f6eee1` = 4.48 — на волосок ниже 4.5; затемнить
  до `#855426`.
- Вторая ссылка «Подробнее →» в карточках дублирует ссылку-заголовок — для
  скринридера вереница одинаковых «Подробнее»; сделать декоративной
  (`aria-hidden`, `tabIndex={-1}`), вся карточка и так кликабельна.
- Alt карточек повторяет заголовок рядом (название звучит дважды) — в карточках
  `alt=""`, на детальных оставить как есть.
- Инлайн-ссылки в тексте отличаются от muted-текста только цветом — тонкое
  постоянное подчёркивание (`text-underline-offset` уже в системе).
- Единый `a:focus-visible, button:focus-visible { outline: 2px solid
  var(--accent); outline-offset: 3px; }` — сейчас у части элементов дефолтный
  браузерный outline (работает, но инородно).
- `target="_blank"` без предупреждения «откроется в новом окне» (строка в
  словарь + aria-label).
- `prefers-reduced-motion` покрывает лайтбокс, но не smooth-scroll
  (`back-to-top.tsx:33`).
- Шрифты в основном в px (78 против 27 rem) — браузерный зум работает, но
  системная настройка «крупный шрифт» px не масштабирует; переводить на rem
  постепенно.
- Тач-мишени: `.lang-switch-link` ≈24×30px — добавить вертикальный паддинг;
  ♡/✓ 40px — осознанно, ок по минимуму WCAG 2.2.
- Карте дать имя: `role="region"` + `aria-label` (`places-map.tsx:134`).
- Кнопка «Поделиться»: «Ссылка скопирована» без live-региона.
- Прогресс OCR в админке меняется в disabled-кнопке без `role="status"`.
- Ошибки админ-форм не связаны с полями (`aria-describedby`/`aria-invalid`);
  textarea OCR-черновика без label.
- Birthdays: в пустом состоянии h1→h3 (пропуск h2).

### Что уже хорошо (доступность) — заметно выше среднего

Семантика без хаков (карточки — растянутая ссылка заголовка, не div-onClick;
чипы — `<button>` с `aria-pressed`); лайтбоксы образцовые (фокус-ловушка,
возврат фокуса, Esc/стрелки, live-счётчик «N из M»); статусы никогда не
«только цветом» («Открыто ещё 3 ч» — всегда текст); `--muted` честно рассчитан
(5.0–5.8:1 — редкость для бежевых палитр); outline нигде не убит, ~28 своих
focus-стилей (видимый фокус подтверждён на проде); поиск — канонический
combobox; `aria-busy` на время ответа сервера; undo вместо безвозвратного
удаления; тайская локаль всерьёз: Sarabun с fallback, буддийская эра,
переведённые aria-строки, защита тайских слов от разрыва; zoom не заблокирован;
пустые состояния с выходом.
